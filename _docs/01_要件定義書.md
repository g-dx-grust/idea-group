## 1. システム全体像

### 1.1 システム構成
```
                ┌────────────────────┐
  ブラウザ/モバイル  │   G-DX (Next.js)    │
  ──────────→  │  ・現場カルテ          │
                │  ・受託簿/進捗        │
  ┌─────────── │  ・見積・請求・計算書 │
  │            │  ・立会申請・帳票     │ ────→ S3 互換 (写真・PDF)
  │            └──────┬────────────┘
  │                   │ REST/OpenAPI
  │ Lark OAuth SSO    ↓
  │            ┌────────────────────┐
  │            │   PostgreSQL         │
  │            └────────────────────┘
  │
  │  （人事・勤怠・経費は Lark 側）
  ↓
┌──────────────────────┐       ┌────────────────┐
│ Lark Base/承認         │ ←→    │ Lark Anycross   │
│  ・勤怠届出            │       │  （双方向同期）  │
│  ・経費精算            │       └────────┬───────┘
└──────────────────────┘                │
                                         ↓ 会計 TXT 生成
                                   ┌──────────┐
                                   │ 経理 PC   │
                                   └──────────┘
```

### 1.2 ロール
| ロール | 主な機能 |
|---|---|
| 土地家屋調査士（社員） | 現場カルテ入力、立会申請、進捗更新、写真登録、勤怠届出、経費精算 |

### 2.4 報酬額計算書モジュール
現行「報酬額計算書入力フォーム」＋「御見積書／御請求書／報酬額計算書」帳票に相当。

#### 2.4.1 データ構造
- 計算書 1 件 = 1 業務（例：筆界確定測量業務）
- 作業明細は以下の 4 区分に分類
  - **調査業務**: 資料調査（公課/地図/図面/縁取書画）、事前調査、現地調査（多角測量/復元測量 基本型/加算型/分筆型 基本型/加算型）、境界確認、民有地（立会・確認／弱音・探索／特殊作業）、公共用地（A/B/C ランク）
  - **測量業務**: 面積測量、境界点設置、境界標設置・移設、引照点測量
  - **申請手続業務**: 各種申請
  - **書類作成等**: 土地証明書・承諾書、土地成果図
- 単価 × 数量 × 加減率 = 報酬額
- 小計① + 小計② + 小計③ + 小計④ = 合計報酬額
- 合計報酬額 − 調整額 = 差引報酬額
- 差引報酬額 + 諸経費 → ×10% 消費税 + 立替金（収入印紙等） = 総合計
- 消費税は「切り捨て（基本）」「四捨五入」を選択可

#### 2.4.2 出力帳票
| 帳票 | 備考 |
|---|---|
| 御見積書 | 見積日付、宛名、所在、業務名、作業種別別金額、合計報酬額、調整額、差引報酬額、諸経費、消費税、立替金、総合計、押印（印影画像） |
| 御請求書 | 見積書と同じ構造＋請求日付、受入口座（振込先） |
| 報酬額計算書 | 明細を作業種別ごとに網羅した一覧表（法定書式） |
| 領収書 | 領収日、領収金額、印紙 |

## 3. 非機能要件

### 3.1 性能
| 指標 | 目標 |
|---|---|
| 画面表示 | 主要画面 < 1.5 秒（95 パーセンタイル） |
| 一覧検索 | 10,000 件中の検索結果 < 2 秒 |
| 帳票生成 | 1 ページ 5 秒以内、一括 30 帳票 60 秒以内 |
| 同時接続 | 50 ユーザー（ピーク） |

### 3.2 可用性
- SLA 99.5%（平日 8〜20 時）
- 障害時の復旧目標 RTO 4 時間、RPO 1 時間

### 3.3 セキュリティ
- Lark SSO + 多要素認証（Lark 側設定）
- HTTPS 必須（TLS1.2 以上）
- DB 保管時暗号化、S3 の SSE 有効
- 個人情報項目のアクセスログ必須
- WAF（Cloudflare）適用

### 3.4 バックアップ
- DB: 日次フル、時間単位 WAL
- 添付ファイル: S3 バージョニング + Cross-region レプリカ
- 保持期間 30 日（通常）、年次アーカイブ 7 年

### 3.5 対応環境
- ブラウザ: Chrome / Edge / Safari 最新 2 バージョン
- モバイル: iOS 16+ / Android 12+（現場向け）
- 印刷: A4 縦／横、両面、モノクロ／カラー

### 3.6 多言語・日付
- 日本語のみ
- 和暦・西暦の両対応（帳票は和暦を基本）

### 3.7 法令・監査
- 電子帳簿保存法（改正後）に準拠した請求書・領収書の保管
- インボイス制度（登録番号を見積・請求書に自動印字）

## 6. 受入基準（抜粋）

- 現行 Access の全業務区分（12 種）の現場登録・更新・一覧・印刷が可能
- 見積・請求・報酬額計算書の金額・内訳が現行と 100% 一致（過去 1 年分の実データで突合）
- 立会申請 30 種以上の帳票が現行と見た目・内容ともに一致
- 勤怠 7 種の届出が Lark 上で入力〜承認まで完結
- 経費精算 TXT が現行会計ソフトでそのまま取り込める
- 並行稼働 2 週間で差異ゼロ（差異があれば原因と補正方法を記録）
