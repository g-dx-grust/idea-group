## 8. 共通 UI ルール

### カラー・ステータス
| ステータス | バッジ色 |
|---|---|
| 進行中 | blue |
| 成果完了 | green |
| 請求済 | yellow |
| 入金済 | emerald |
| 終了 | gray |
| 期限超過 | red |

### バリデーション
- 保存時に必須フィールド検証（`zod` スキーマ）
- 日付の前後関係（開始 < 終了）
- 報酬額計算: 整数のみ、加減率 0〜200%

### 操作フロー
- **下書き保存**: バリデーションなし、即保存
- **保存**: バリデーション後、`updatedAt` 更新
- **削除**: 論理削除（`deletedAt` セット）、確認ダイアログ必須
- **帳票生成**: 非同期（BullMQ）、生成中はローディング、完了後 PDF を新タブで開く

### モバイル対応
- タブナビは横スクロール（`overflow-x-auto`）
- テーブルは縦並びカード形式に切替（`< md`）
- 日付入力は `<input type="date">` でネイティブピッカー活用
